各种引用类型的操作方法-2

发表于 2017-11-15 | 分类于 web开发 ， api大全

Object.isExtensible() 和 Object.isFrozen() 和 Object.isSealed()

Object.isExtensible() 方法判断一个对象是否是可扩展的（是否可以在它上面添加新的属性）。
注意：默认情况下，对象都是可以扩展的，即可以为它们添加新的属性。以及它们的proto可以被更改。Object.preventExtensions，Object.seal 或 Object.freeze 方法都可以标记一个对象为不可扩展（non-extensible）。

Object.isFrozen()方法判断一个对象是否被冻结。一个对象是冻结的是指它不可扩展，所有属性都是不可配置的，且所有数据属性（即没有getter或setter组件的访问器的属性）都是不可写的。

Object.isSealed() 方法判断一个对象是否被密封。密封对象是指那些不可扩展的，且所有自身属性都不可配置且因此不可删除（但不一定是不可写）的对象。

Object.prototype.hasOwnProperty()

obj.hasOwnProperty(prop) 方法会返回一个布尔值，指示对象是否具有指定的属性作为自身（不继承）属性。

Object.prototype.isPrototypeOf()

prototypeObj.isPrototypeOf(obj) 方法用于测试一个对象是否存在于另一个对象的原型链上。
注意：isPrototypeOf() 与 instanceof 运算符不同。在表达式 “object instanceof AFunction”中，object 的原型链是针对 AFunction.prototype 进行检查的，而不是针对 AFunction 本身。

Object.prototype.propertyIsEnumerable()

obj.propertyIsEnumerable(prop) 方法返回一个布尔值，表示指定的属性是否可枚举。

Object.prototype.toLocaleString()

obj.toLocaleString() 方法调用每个数组元素的 toLocaleString() 方法，然后使用地区特定的分隔符把生成的字符串连接起来，形成一个字符串。
obj.toString()方法获取的是String(传统字符串),而toLocaleString()方法获取的是LocaleString(本地环境字符串)。
注意： 使用toString()是保险的，返回唯一值的方法,它不会因为本地环境的改变而发生变化。如果是为了返回时间类型的数据，推荐使用LocaleString()。若是在后台处理字符串，请务必使用toString()。

Object.prototype.toString()

obj.toString() 方法返回一个表示该对象的字符串。toString() 方法被每个Object对象继承。如果此方法在自定义对象中未被覆盖，toString() 返回 “[object type]”，其中type是对象的类型。

Object.prototype.valueOf()

obj.valueOf() 方法返回指定对象的原始值。JavaScript调用valueOf方法将对象转换为原始值。你很少需要自己调用valueOf方法；当遇到要预期的原始值的对象时，JavaScript会自动调用它。如果对象没有原始值，则valueOf将返回对象本身。

Object.setPrototypeOf()

Object.setPrototypeOf(obj, prototype) 方法设置一个指定的对象的原型 ( 即, 内部[[Prototype]]属性）到另一个对象或null。
注意： 更改对象的 [[Prototype]]在各个浏览器和 JavaScript 引擎上都是一个很慢的操作。其在更改继承的性能上的影响是微妙而又广泛的，这不仅仅限于 obj.proto = … 语句上的时间花费，而且可能会延伸到任何代码，那些可以访问任何[[Prototype]]已被更改的对象的代码。如果你关心性能，你应该避免设置一个对象的 [[Prototype]]。相反，你应该使用 Object.create()来创建带有你想要的[[Prototype]]的新对象。

Object.values()

Object.values(obj) 方法返回一个给定对象自己的所有可枚举属性值的数组，值的顺序与使用for…in循环的顺序相同 ( 区别在于 for-in 循环枚举原型链中的属性 )。

各种引用类型的操作方法-1

发表于 2017-11-09 | 分类于 web开发 ， api大全

概括

计划把es5,es6中的所有引用类型(Object, Array, Date, RegExp, Function, Boolean, Number, String, Global, Math)的各种操作方法给上手总结下，熟悉和理解。

Object

Object构造函数

Object构造函数为给定值创建一个对象包装器。如果给定值是null或undefined，将会创建并返回一个空对象，否则，将返回一个与给定值对应类型的对象。

Object.assign()

Object.assign(target, …sources) 方法用于将所有可枚举属性的值从一个或多个源对象复制到目标对象。它将返回目标对象。

Object.create()

Object.create(proto[, propertiesObject]) 方法会使用指定的原型对象及其属性去创建一个新的对象。

Object.defineProperties()

Object.defineProperties(obj, props) 方法直接在一个对象上定义新的属性或修改现有属性，并返回该对象。

Object.entries()

Object.entries(obj) 方法返回一个给定对象自身可枚举属性的键值对数组，其排列与使用 for…in 循环遍历该对象时返回的顺序一致（区别在于 for-in 循环也枚举原型链中的属性）。

Object.freeze()

Object.freeze(obj) 方法可以冻结一个对象，冻结指的是不能向这个对象添加新的属性，不能修改其已有属性的值，不能删除已有属性，以及不能修改该对象已有属性的可枚举性、可配置性、可写性。也就是说，这个对象永远是不可变的。该方法返回被冻结的对象。

Object.getOwnPropertyDescriptor()

Object.getOwnPropertyDescriptor(obj, prop) 方法返回指定对象上一个自有属性对应的属性描述符。(自有属性是指直接赋予该对象的属性，不需要从原型链上查找的属性。)
Object.getOwnPropertyDescriptors(obj) 方法用来获取一个对象的所有自身属性的描述符。

Object.getOwnPropertyNames() 和 Object.key() 和 for..in..的对比

Object.getOwnPropertyNames(obj) 方法返回一个由指定对象的所有自身属性的属性名（包括不可枚举属性但不包括Symbol值作为名称的属性）组成的数组。
Object.key(obj) 方法会返回一个由一个给定对象的自身可枚举属性组成的数组，数组中属性名的排列顺序和使用 for…in 循环遍历该对象时返回的顺序一致 （两者的主要区别是 一个 for-in 循环还会枚举其原型链上的属性）。

Object.getOwnPropertySymbols()

Object.getOwnPropertySymbols() 方法返回一个给定对象自身的所有 Symbol 属性的数组。
注意： 与Object.getOwnPropertyNames()类似，您可以将给定对象的所有符号属性作为 Symbol 数组获取。 请注意，Object.getOwnPropertyNames()本身不包含对象的 Symbol 属性，只包含字符串属性。因为所有的对象在初始化的时候不会包含任何的 Symbol，除非你在对象上赋值了 Symbol 否则Object.getOwnPropertySymbols()只会返回一个空的数组。

Object.getPrototypeOf()

Object.getPrototypeOf(obj) 方法返回指定对象的原型（内部[[Prototype]]属性的值）。

Object.is()

Object.is(value1, value2) 方法判断两个值是否是相同的值。
Object.is() 判断两个值是否相同。如果下列任何一项成立，则两个值相同：
两个值都是 undefined
两个值都是 null
两个值都是 true 或者都是 false
两个值是由相同个数的字符按照相同的顺序组成的字符串
两个值指向同一个对象
两个值都是数字并且 都是正零 +0, 都是负零 -0, 都是 NaN, 都是除零和 NaN 外的其它同一个数字

13. 配置ETag

(1) Last-Modified 和 ETag 是协商缓存，返回304，前面规则2有提到过。
(2) 没有强缓存时使用协商缓存。
(3) 服务器通过Last-Modified响应头来返回组件的最新修改日期，浏览器使用If-Modified-Since头将最新修改日期传回到原始服务器进行比较，如果服务器上组件的最新修改日期和浏览器传回的值匹配，则返回304 Not Modified。
(4) ETag 检测浏览器缓存中的组件与原始服务器上的组件是否匹配，它比验证修改日期更加灵活，优先级也更高。浏览器使用If-None-Match头将一串标识符传回服务器进行匹配。

14. 使Ajax可缓存

高性能网站建设指南-4

发表于 2017-11-07 | 分类于 web开发 ， 前端性能优化

10. 精简JavaScript

(1) 精简：移除不必要的字符(空格，换行和制表符)和所有的注释来减少文件的大小。
(2) 混淆：除了精简外，它还会改写代码，函数和变量的名字将被转换为更短的字符串。
(3) 精简脚本可以降低响应时间，但是不会带来混淆的风险。
(4) 脚本可以通过gzip压缩后再用其它工具精简。

11. 避免重定向

(1) 重定向中301, 302是最常用的两种，重定向会使你的页面变慢。
(2) 重定向延迟了整个HTML文档的传输。

12. 删除重复脚本

团队越多，脚本数量越大，产生重复脚本的概率越大。对于高概率被引用的脚本，通过包管理系统，确保脚本只被包含一次。

高性能网站建设指南-3

发表于 2017-11-06 | 分类于 web开发 ， 前端性能优化

7. 避免CSS表达式

(1) css expression方法是IE中设置动态CSS属性的一种强大方式。
(2) 表达式的问题在于对其进行求值的频率比人们期望的要高。
(3) CSS3 中的calc()是动态求width的函数，可以安全使用。

8. 使用外部JavaScript和CSS

(1) 纯粹来说，外部组件要承担更多的HTTP请求带来的开销。
(2) 外部组件的收益: 有机会被浏览器缓存下来。内联组件由于被包含在动态的HTML中，通常不会被配置为可以进行缓存。
(3) 每个用户产生的页面浏览量越少，内联组件表现的越好。
(4) 如果网站的每个页面中都使用了相同的JavaScript和CSS，使用外部文件可以提高文件的重用率。

9. 减少DNS查找

(1) 很多浏览器都有自己的DNS缓存，和操作系统的缓存相分离，只有当浏览器丢弃了记录后它才会向操作系统询问地址。
(2) 服务器可以表明记录可以被缓存多久，查找返回的DNS记录包含了一个存活的时间(TTL)。操作系统会考虑TTL，但是浏览器通常忽略该值，并设置它自己的时间限制。
(3) Keep-Alive会覆盖TTL和浏览器的时间限制，意思是只要TCP保持打开，就不会进行DNS查找。
(4) 当客户端没有DNS缓存时，DNS查找的数量和WEB页面中唯一主机名的数量相等，所以减少唯一主机名的数量就可以减少DNS查找的数量。
(5) PC端采用域名发散，增加主机名的数量，提高页面的并行下载数量。
(6) 移动端采用域名收敛，减少主机名的数量，提高DNS的查找速度。
(7) DNS查找的时间和并行下载的时间需要仔细权衡。

高性能网站建设指南-2

发表于 2017-11-05 | 分类于 web开发 ， 前端性能优化

4. 压缩组件

(1) 该规则通过减少HTTP响应的大小来减少响应时间。
(2) web客户端可以通过HTTP请求中的Accept-Encoding头来表示对压缩的支持。

Request Headers: Accept-Encoding: gzip, deflate

Response Headers: Content-Encoding: gzip
(3) 图片和PDF不应该压缩，它们本来就被压缩了。试图对其压缩不仅耗费CPU的资源，还可能增加文件大小。
(4) 压缩的成本———服务器会花费额外的CPU周期完成压缩，客户端还得花时间解压。

5. 将样式表放在顶部

(1) 在浏览器和用户等待底部的样式表时，浏览器会延时显示任何可视化组件，这种现象叫做”白屏”。
(2) 样式表导入的规则：link标签和@import规则，@import规则在所有规则之前才能生效，并且会导致组件下载时的无序性。
(3) 样式表在页面中的位置不影响下载时间，但是会影响页面的呈现。

6. 将脚本放在底部

(1) HTTP1.1规范中建议每个主机名并行下载两个组件，但是工程师和用户可以自行修改配置。
(2) 增加并行下载数量不是没有开销的：取决于你的带宽和CPU速度，过多的并行下载反而会降低性能。
(3) 脚本不能并行下载，其一是脚本可能改变页面内容，其二是保证脚本能够按照正确的顺序执行。
(4) 将脚本放在顶部会导致: 阻塞后面页面的呈现 和 阻塞后面组件的下载。
(5) 如果一个脚本可以延迟，那么它一定可以移动到页面的底部。

高性能网站建设指南-1

  • 这篇博客是我最近看的<<高性能网站建设指南>>一书中提出的14条优化规则加上我对其的理解整理而来。
  • 所有规则的案例可以在http://stevesouders.com/hpws中找到并测试效果。
  • 着重在network中观察Status, Time, Size, Cache-Control, Content-Encoding, Etag, Last-Modified 和 Keep-Alive, 这些条目提供了很好的参数供我们去理解前端的性能。
  • 建议使用firefox测试，我在chrome中发现少数地方(比如强缓存)中存在难以理解的行为:)

1. 减少http请求

图片地图

(1) 图片地图允许你在一个图片上关联多个URL, 我们通常使用HTML中的MAP标签实现。
(2) 缺点：手工方式定义图片上的坐标，而且除了矩形之外几乎无法定义其他形状。
(3) 导航栏或者多个超链接中的背景可以使用图片地图。

CSS Sprites

(1) 将多幅图片合并为一副单独的图片(一般是图标矢量图)。
(2) 通过background-position指定偏移量来取得图片区域。

内联图片

(1) 通过data:URL模式可以在web页面中包含图片但无需任何额外的HTTP请求。
(2) 图片的数据就在该URL自身之中,其格式为: data:[<mediatype>] [;base64],<data>
(3) data:URL是内联在页面中的，在跨越不同页面时不会被缓存。
(4) 该URL通过后端语言对图片编码输出，PHP中为base64_encode()。

合并脚本和样式表

使用webpack打包[模块化]的组件

2. 使用内容发布网络(CDN)

如果web服务器离用户的距离更近，那么一个http请求的响应时间就会缩短。

3. 添加Expires头

强缓存

(1) 在HTTP1.0中web服务器使用Expires头告知web客户端它可以使用一个组件的当前副本，直到指定时间为止。Response Headers: Expires: Mon, 15 Apr 2024 20:00:00 GMT
(2) 在HTTP1.1中引入了Cache-Control头来克服Expires头的限制，因为Expires要求服务器和客户端的时间严格同步，过期时间还要经常检查， 过期后还要重新配置。
(3) Cache-Control使用max-age指定组件被缓存多久，Cache-Control会覆盖Expires，
Response Headers: Cache-Control: max-age=315360000
(4) 强缓存后再请求该组件，浏览器将会直接从缓存中读取(from memory cache, from disk cache)，虽然状态码仍为200,但是浏览器并没有产生http请求。
(5) 如果在缓存期间组件需要更新，可以通过修改组件名的方法让用户获得最新的组件。一般来说，可以把版本号嵌入组件名中，比如：example_v1.js。

协商缓存

(1) Response Headers: Last-Modified or Etag
(2) Request Headers: If-Modified-Since or If-None-Match
(3) 协商缓存返回状态码304(not modified)
(4) 详细内容在后面的规则会提到=.=

8. Language attribute

(1) lang 属性规定元素内容的语言。
(2) HTML5中强烈建议为 html 根元素指定 lang 属性，从而为文档设置正确的语言。这将有助于语音合成工具确定其所应该采用的发音，有助于翻译工具确定其翻译时所应遵守的规则等等。

(2) 用了Meta Property=og标签，就是你同意了网页内容可以被其他社会化网站引用等，目前这种协议被SNS网站如Fackbook、renren采用。任何网页只要遵守该协议，SNS就能从页面上提取最有效的信息并呈现给用户。
